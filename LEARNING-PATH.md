# 2026 Vibe Coder 12周学习路径：从零到编排者

> **目标读者**：完全不会编程的人，想通过 AI 工具（Claude Code、Cursor、Replit Agent 等）构建软件产品
> **总时长**：12 周，每周投入 5-8 小时
> **核心理念**：你不需要学会写代码，你需要学会**指挥 AI 写代码**
> **设计日期**：2026年2月

---

## 使用说明

### 本文档是什么

这是一份将以下资源有机融合的**完整学习计划**：

| 来源 | 内容 | 标记方式 |
|------|------|---------|
| CURRICULUM.md | 7个主线项目（P1-P7） | `[主线]` |
| CURRICULUM-EXTENDED.md | 6个拓展项目（E1-E6）+ 品味挑战 | `[拓展]` |
| TASTE-CHALLENGES.md | 153个品味挑战（A1-L8） | 直接引用编号，如 `A1`、`B3` |
| TASTE-EXERCISES.md | 25个架构深度练习 | `[深度练习]` |
| 06-ai-taste-exercises.md | 28个AI/提示工程练习 | `[AI练习]` |
| taste-building-exercises-cn.md | 30个UX/产品练习 | `[产品练习]` |
| marketing-business-taste-exercises.md | 30个营销/商业练习 | `[营销练习]` |

### 如何使用

1. **按周推进**：每周有明确的学习目标、项目实践、品味挑战和每日微练习
2. **不要跳过品味挑战**：它们不是"选做"，而是和项目同等重要的核心练习
3. **每日微练习**：每天5分钟，雷打不动。品味是靠日积月累练出来的
4. **里程碑验证**：每周末检查里程碑清单，诚实地评估自己
5. **灵活调整**：如果某周内容没完成，允许延长，但不要跳过

### 四个阶段概览

```
第1-3周：观察者    — 建立基本认知，理解互联网和软件是什么
第4-6周：实践者    — 开始动手做项目，用 AI 构建真实产品
第7-9周：评论家    — 能判断方案好坏，能写出好的规格说明
第10-12周：创造者  — 能独立设计方案，编排 AI 完成复杂任务
```

## 第1周：认识软件世界

### 周目标

- 理解互联网如何工作（浏览器请求 → 服务器响应 → 渲染页面）
- 理解前端 vs 后端、静态 vs 动态的区别
- 安装并熟悉 AI 编码工具（Claude Code 或 Cursor）
- 完成第一次"把东西放到互联网上"的体验

### 项目实践

**[主线] Project 1：「数字名片」— 个人品牌站（第一部分）**

本周只完成以下步骤：
1. 安装 Claude Code 或 Cursor，理解终端基础
2. 用 AI 生成一个简单的个人页面（名字 + 自我介绍 + 联系方式）
3. 理解文件和文件夹的组织方式
4. 连接 GitHub，完成第一次推送
5. 部署到 Vercel，让全世界都能访问

**关键知识点**（来自 CURRICULUM.md 第一部分和第二部分）：
- 阅读"从 Vibe Coding 到 Agentic Engineering"的演变轨迹
- 理解四层能力栈：基础层 → 通信层 → 治理层 → 编排层
- 理解"Just Works"入门技术栈

### 品味挑战（本周选做3个）

| 编号 | 挑战 | 时间 | 为什么本周做 |
|------|------|------|------------|
| **B1** | 5秒测试 | 30分钟 | 打开5个你常用的应用，每个看5秒后闭眼回答：它是做什么的？主要功能在哪？ |
| **B10** | 模糊测试（Squint Test） | 30分钟 | 眯眼看界面，训练视觉层次感知 |
| **B15** | 色彩心理学解码 | 30分钟 | 分析5个品牌的色彩策略，理解颜色如何影响感觉 |

### 每日微练习（每天5分钟）

| 周一 | 周二 | 周三 | 周四 | 周五 | 周末 |
